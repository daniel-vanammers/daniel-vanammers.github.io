HTML and CSS as a Simple Analogy

We can use a car for an HTML and CSS Analogy:

HTML brings the car into existence by creating the elements that give it its form - it has an exterior, windows, doors, 4 wheels and other bits and pieces that one would generally expect in a car.

CSS comes along and describes the form of the car - it is gun metal grey, it has a leather steering wheel, it has alloy wheels.

JavaScript gives the car it's functionality. It has windows that open and close, it has a stereo that can play music, it has buttons that the user can push to elicit outputs and the car can move from A to B.

Control Flow and Loops

Control flow is simply the order in which code is read - generally from the top of a page to the bottom. We can use loops to streamline the execution of repetitive tasks. For example:

Making an Omelette:

  1. Heat butter in a pan
  2. Crack an egg into a bowl
  3. Crack another egg into a bowl
  4. Crack another egg into a bowl
  5. Crack another egg into a bowl
  6. Beat the eggs
  7. Cook your omelette

A loop would shorten this set of instructions to just 4 lines of code:

  1. Heat butter in a pan
  2. Crack each egg you are using into a bowl
  3. Beat the eggs
  4. Cook your omelette

This makes your code less verbose, less prone to error and easier to read.

What is the DOM?

Your browser displays a web document that is a combination of HTML, CSS and JavaScript. Underneath these three languages is a system that your browser uses to make sense of it all. This structure is known as the Document Object Model (DOM).

The DOM organises your document into a tree-like structure with each element a separate node. We can then use a language like JavaScript to independently access different elements at these nodes.

We use Developer Tools that are built into most browsers to access and manipulate our elements.

Arrays and Objects

Firstly, what is an array? An array basically helps us store, organise, manipulate and retrieve a collection of data. For example:

milkTypes = [“Soy”, “Almond”, “Moo Cow”, “Hemp”];

Each of the strings in an array has an index, which starts at the number zero and works up in increments of one [0,1,2,3,4] etc. We can access the data using their indexes:

milkTypes[2] will access the data point “Moo Cow”.

Almost everything in JavaScript is an object, which can be considered a collection of named values. For example:

var thisGuyRightHere = {firstName: “Dan”,
                        lastName: “van Ammers”,
                        age: 31,
                        eyeColor: “blue”};

The variable “thisGuyRightHere” points to an object with a selection of different named values, with information after the colons as the properties. We can access these properties in two ways. Either with dot notation like so:

thisGuyRightHere.lastName which will return the property “van Ammers”.

Or with bracket notation as follows:

thisGuyRightHere[“age”] which will return the property 31.

Functions!

Functions are one of the fundamental building blocks in JavaScript. A function is a set of statements that performs a defined task and returns a result to the user. The idea is to make a function that will streamline a repetitive task to save you from writing the same code again and again. This serves the dual purpose of saving you time and avoiding needlessly verbose code.

An easy example is a loop function that removes the necessity of repeating duplicate steps to deliver a result.
